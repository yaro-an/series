# Сериалы

Локальное приложение для списка сериалов с оценками, комментариями и обложками.

## Запуск

Дважды кликни по `Запустить Сериалы.command`, чтобы открыть приложение через локальный адрес.

Для версии с пользователями, логином, ролями и общей серверной базой запускай `Запустить сервер.command`.

Данные сохраняются в локальном хранилище браузера. В приложении есть экспорт и импорт JSON-файла для резервной копии.

Обложки лежат локально в `assets/covers`. Описания, актёры, сезоны и серии лежат в `assets/show-info.json`, русские переводы описаний — в `assets/show-info-ru.json`. Данные скачаны через TVMaze API, переводы сохранены локально.

В приложении есть статусы просмотра, режим постерной полки и кнопка трейлера в подробностях сериала. Разделы автоматически синхронизируются со статусом просмотра.

При добавлении нового сериала приложение автоматически пробует загрузить афишу, описание, сезоны, серии и актёров через TVMaze. Для этого в момент добавления нужен интернет; найденные данные сохраняются в браузере.

Хранение данных перенесено в локальную базу браузера IndexedDB. Старый `localStorage` остаётся как резервный слой и для автоматической миграции старых записей.

Боковую панель можно сворачивать кнопкой меню в левом верхнем углу. Интерфейс обновлён в более стеклянном liquid-glass стиле.

## Пользователи

Серверная версия использует `server.js` и базу `data/app-db.json`. Админ может входить, смотреть список пользователей и выдавать права на добавление или редактирование сериалов. Пароли пользователей хранятся как PBKDF2-хеши, не открытым текстом.

Обычный статический запуск остаётся без настоящей авторизации: для ролей и общей базы нужен именно серверный запуск.

## Публикация по ссылке

Для версии с пользователями проект нужно публиковать как Node.js web service, например на Render. Netlify Drop / GitHub Pages подходят только для статической версии без настоящего логина.

### Render

Проект подготовлен для Render:

- `package.json` — команда запуска `npm start`
- `render.yaml` — Blueprint для Render
- `server.js` — Node.js сервер
- `data/app-db.json` — стартовая база

Как публиковать:

1. Создай GitHub-репозиторий и загрузи туда проект.
2. В Render выбери New -> Blueprint или New -> Web Service.
3. Подключи репозиторий.
4. Build command: `npm install`.
5. Start command: `npm start`.
6. Health check path: `/api/health`.
7. Render даст ссылку вида `https://...onrender.com`.

`render.yaml` настроен на paid `starter` web service с persistent disk `/var/data`, потому что файл-база должна сохраняться между рестартами. На free web service без диска база может сбрасываться при redeploy/restart.

После деплоя админ входит через серверную форму входа. Обычные пользователи регистрируются сами, а админ выдаёт им права в панели `Пользователи`.

В `index.html`, `robots.txt`, `netlify.toml` и `vercel.json` включён `noindex`, чтобы поисковики не добавляли сайт в выдачу. Это не парольная защита: сайт будет доступен всем, у кого есть ссылка и аккаунт.
